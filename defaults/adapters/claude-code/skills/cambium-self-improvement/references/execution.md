# Execution

Identify self-improvement tasks by their description prefix. For upstream merge/contribution tasks, use the `cambium-update` and `cambium-contribute` skills instead.

## SELF-IMPROVEMENT TASK 1 — Write eval + baseline

Find the repo root first: `REPO_DIR=$(git rev-parse --show-toplevel)`

1. **Parse the task description** for: target file, observation, proposed change, evidence paths
2. **Read evidence** — examine the session digests listed as evidence
3. **Query recent episodes** for representative trigger messages:
   ```bash
   curl -s "$CAMBIUM_API_URL/episodes?since=$(date -u -v-7d +%Y-%m-%dT%H:%M:%SZ)&until=$(date -u +%Y-%m-%dT%H:%M:%SZ)&routine=coordinator&limit=10"
   ```
4. **Write the eval config** to `evals/<target-slug>-<date>.yaml`:
   - Choose a scenario that exercises the target behavior
   - Use representative trigger messages as injection payloads
   - Include assertions that measure the behavior described in the observation
   - Set `trials: 3`, timeout 120-180s (single routine) or 300s (cascade)
5. **Write the config override** to `evals/<target-slug>-<date>.override.yaml`:
   - For prompt changes: use `append`, `content`, or `patch` mode
   - For YAML changes: specify the key-value overrides
6. **Run the baseline eval** (no override):
   ```bash
   cd "$REPO_DIR"
   .venv/bin/python -m cambium eval evals/<name>.yaml \
     --save-baseline baselines/<name>.json \
     --output json --repo-dir "$REPO_DIR"
   ```
7. **Commit** eval config + override + baseline to the repo
8. **Store paths in work item context**:
   ```bash
   curl -s -X PATCH "$CAMBIUM_API_URL/work-items/ITEM_ID/context" \
     -H 'Content-Type: application/json' \
     -H "Authorization: Bearer $CAMBIUM_TOKEN" \
     -d '{
       "eval_path": "evals/<name>.yaml",
       "override_path": "evals/<name>.override.yaml",
       "baseline_path": "baselines/<name>.json",
       "target_file": "<target-file>",
       "observation": "<observation>"
     }'
   ```
9. **Complete** with a summary of the eval config and baseline results.

## SELF-IMPROVEMENT TASK 2 — Test change + create PR

1. **Read work item context** from the parent or sibling for `eval_path`, `override_path`, `baseline_path`, `target_file`, `observation`.
2. **Run canary eval** — verify the change doesn't break the system:
   ```bash
   cd "$REPO_DIR"
   .venv/bin/python -m cambium eval defaults/evals/canary-cascade.yaml \
     --config-override "$override_path" --repo-dir "$REPO_DIR"
   ```
   If canary fails: fail the task with "Canary eval failed: proposed change breaks system integrity."
3. **Run comparison eval**:
   ```bash
   .venv/bin/python -m cambium eval "$eval_path" \
     --config-override "$override_path" \
     --compare-baseline "$baseline_path" --repo-dir "$REPO_DIR"
   ```
4. **Evaluate**: improved or maintained → proceed to PR. Regressed → fail with details.
5. **Create PR**:
   ```bash
   BRANCH="improve/$(echo "$target_file" | tr '/' '-' | sed 's/\..*//g')-$(date +%Y%m%d)"
   git worktree add "/tmp/cambium-improve-$$" -b "$BRANCH"
   cd "/tmp/cambium-improve-$$"
   # Apply the actual file change, commit, push
   git push -u origin "$BRANCH"
   gh pr create --title "Self-improvement: <description>" --label self-improvement \
     --body "## Self-Improvement Proposal
   **Observation:** <observation>
   **Target:** <target_file>
   ## Eval Results
   - **Baseline:** <X>%
   - **With change:** <Y>%
   ## Evidence
   <evidence list>
   ---
   *Automatically generated by the Cambium self-improvement loop.*"
   # Cleanup
   cd "$REPO_DIR" && git worktree remove "/tmp/cambium-improve-$$" --force
   ```
6. **Complete** with the PR URL and eval comparison summary.

## Failure modes

- **Eval syntax gap**: Cannot express needed test → fail with explanation. Planner creates implementation task to extend eval framework.
- **Canary failure**: Change is too risky → fail immediately.
- **Regression**: Do not create PR → fail with comparison details.
