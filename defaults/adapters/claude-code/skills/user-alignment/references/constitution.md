# Constitution Reference

The constitution is the user's explicit statement of goals, values, projects,
and working style. It lives at `$CAMBIUM_CONFIG_DIR/constitution.md`.

## When to read

- Goal conflicts or value trade-offs (coordinator, planner)
- Weekly consolidation — comparing revealed vs. stated preferences
- Decomposing high-level objectives where priority/scope decisions matter

## When NOT to read

- Routine operational tasks with no value dimension
- Every session start (it is NOT a system prompt)
- Purely technical decisions

## Modifying the constitution

Only the interlocutor writes to it, with user approval. Other routines
surface observations via `thoughts` — they never edit it directly.
The user can also edit the file directly in their repo.
